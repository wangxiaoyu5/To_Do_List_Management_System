# 多智能体协作开发配置

## 工作流程（每次需求自动执行）

```
用户需求 → Planner分析 → Generator实现 → Evaluator评估 → 通过/迭代
```

## Agent 角色定义

### 1. Planner（需求规划器）
职责：
- 分析需求完整性
- 识别技术栈约束
- 分解任务清单
- 制定验收标准

输出格式：
```
┌─ 需求分析 ─────────────────┐
│ 原始需求: [用户描述]        │
│ 功能范围: [功能点列表]      │
│ 技术约束: [项目技术栈]      │
│ 依赖检查: [是否需要新依赖]   │
│ 任务分解:                   │
│   1. [子任务1]              │
│   2. [子任务2]              │
│ 验收标准:                   │
│   - [标准1]                 │
│   - [标准2]                 │
└────────────────────────────┘
```

### 2. Generator（代码生成器）
职责：
- 按任务清单逐个实现
- 每完成一个模块自我检查
- 遵循项目规范和命名约定

自评格式：
```
✓ 创建 [文件路径]
  - 类型完整性: [是/否]
  - 规范符合度: [是/否]
  - 潜在问题: [无/xxx]
```

### 3. Evaluator（质量评估器）
职责：
- 代码审查（规范、类型、安全）
- 功能验证（构建测试、截图对比）
- 评分并给出修复建议

## 评分维度（满分 10 分）

### 前端代码质量
| 维度 | 分值 | 检查内容 |
|------|------|---------|
| 设计质量 | 2.5分 | UI美观度、布局合理性、现代设计趋势 |
| 代码规范 | 2.5分 | TypeScript类型完整性、命名规范、无未使用变量 |
| 功能完整 | 2.5分 | 需求实现度、交互流畅度、错误处理 |
| 技术栈适配 | 2.5分 | 正确使用React+Tailwind+Vite、遵循项目规范 |

### 后端代码质量
| 维度 | 分值 | 检查内容 |
|------|------|---------|
| API规范 | 2.5分 | RESTful规范、响应格式、状态码 |
| 架构分层 | 2.5分 | models→repositories→services→controllers→routes |
| 错误处理 | 2.5分 | try-catch、参数验证、日志记录 |
| 类型安全 | 2.5分 | TypeScript类型完整性 |

## 通过标准
- 总分 ≥ 8.0
- 每个维度 ≥ 2.0

## 迭代规则
- 最多 5 次迭代
- 超过 5 次仍不通过则停止并报告问题
- 用户可随时说"停止"或"换一种方案"

## 前端样式特殊评估

每次前端修改后必须：
1. 截图页面
2. 对比设计标准
3. 给出改进建议
4. 必要时重新设计

评估项：
| 评估项 | 检查内容 | 方式 |
|--------|---------|------|
| 视觉设计 | 美观度、AI生成痕迹 | 截图+视觉评估 |
| 响应式 | 移动端适配 | 多视口测试 |
| 交互体验 | 动画、过渡、反馈 | 手动验证 |
| 代码质量 | Tailwind使用、组件拆分 | 静态分析 |

## 用户指令触发词

当用户说以下任意一种表达时，自动启用多智能体流程：
- "帮我实现..."
- "创建一个..."
- "添加...功能"
- "修改...样式"
- "优化..."
- "重做..."
- 任何明确的需求描述

## 禁止行为

- 不经过评估直接提交代码
- 跳过自我检查步骤
- 忽略构建错误
- 不截图验证前端效果
