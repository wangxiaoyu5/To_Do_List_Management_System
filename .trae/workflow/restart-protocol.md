# 重启协议

## 1. 重启前必读

### 1.1 需要读取的文件清单

重启前必须读取以下文件以了解项目状态：

```
📋 必读文件
├── .trae/
│   ├── agent/
│   │   ├── README.md              # Agent规范总览
│   │   └── permissions.md         # 权限边界
│   ├── rules/
│   │   ├── README.md              # 规范总览
│   │   ├── frontend.md            # 前端规范
│   │   ├── backend.md             # 后端规范
│   │   ├── database.md            # 数据库规范
│   │   └── git.md                 # Git规范
│   └── workflow/
│       ├── permission-policy.md   # 权限策略
│       ├── restart-protocol.md    # 本文件
│       └── verify-checklist.md    # 验证清单
├── 需求文档.md                     # 项目需求
├── 开发进度计划.md                 # 开发计划
└── .trae/workflow/progress/
    └── current-task.md            # 当前任务（如果存在）
```

### 1.2 快速状态检查

```
1. 项目是否已有代码？
2. 当前任务是什么？
3. 上次做到哪一步？
4. 是否有未完成的工作？
```

## 2. 重启步骤

### 2.1 步骤1：环境验证

#### 检查项目结构

```bash
# 检查是否存在以下目录
- frontend/
- backend/
- docs/
- .trae/
```

#### 检查关键文件

```
[ ] package.json（前端）
[ ] requirements.txt（后端）
[ ] manage.py（后端）
[ ] .env.example
[ ] prisma/schema.prisma（如果有）
```

### 2.2 步骤2：依赖检查

#### 前端依赖检查

```
[ ] node_modules/ 是否存在
[ ] package.json 是否存在
[ ] npm run build 是否可用
[ ] npm run type-check 是否可用
[ ] npm run lint 是否可用
```

#### 后端依赖检查

```
[ ] Python 环境
[ ] virtualenv/venv 是否存在
[ ] requirements.txt 是否完整
[ ] django-admin 是否可用
```

### 2.3 步骤3：数据库检查

```
[ ] 数据库配置是否存在
[ ] Prisma schema 是否存在（如果使用）
[ ] 迁移文件是否完整
[ ] .env 中的数据库连接配置（仅读取结构，不读取值）
```

### 2.4 步骤4：代码验证

```
[ ] 检查是否有未完成的修改
[ ] 检查是否有语法错误
[ ] 运行 GetDiagnostics 检查
[ ] 查看是否有明显问题
```

## 4. 状态恢复流程

### 4.1 场景1：继续任务

**当存在 current-task.md 时：**

```
1. 读取 .trae/workflow/progress/current-task.md
2. 了解当前任务状态
3. 查看已完成、进行中、待处理的事项
4. 从上次停止的地方继续
5. 更新任务进度
```

### 4.2 场景2：完成任务

**当任务实际上已完成但没有记录时：**

```
1. 验证任务确实已完成
2. 运行相关检查
3. 调用代码审查
4. 创建任务完成记录
5. 准备开始下一个任务
```

### 4.3 场景3：处理失败验证

**当验证发现问题时：**

```
1. 记录发现的问题
2. 评估问题严重程度
3. 询问用户如何处理
4. 根据用户指示执行
5. 确保问题解决后再继续
```

### 4.4 场景4：全新开始

**当没有 current-task.md 或项目全新时：**

```
1. 阅读需求文档
2. 阅读开发进度计划
3. 询问用户从哪里开始
4. 创建新的 current-task.md
5. 开始第一个任务
```

## 5. 重启后立即执行

### 5.1 立即检查清单

```
□ 读取了所有必读规范文件

```

## 6. 常见问题解答

### Q1: 如果找不到 current-task.md 怎么办？

**A**: 
```
1. 检查项目根目录是否有相关文档
2. 询问用户当前在做什么
3. 创建新的 current-task.md
4. 从用户确认的地方开始
```

### Q2: 如果发现代码有错误怎么办？

**A**:
```
1. 不要立即修复
2. 记录错误
3. 向用户报告
4. 询问是否需要修复
5. 根据用户指示行动
```

### Q3: 如果环境不完整怎么办？

**A**:
```
1. 列出缺少的部分
2. 评估影响
3. 向用户报告
4. 询问是先设置环境还是先做其他
5. 按用户指示执行
```

### Q4: 如果用户要求做规范禁止的事怎么办？

**A**:
```
1. 引用相关规范
2. 解释为什么禁止
3. 提供替代方案
4. 如果用户坚持，记录并按权限升级流程处理
```

### Q5: 重启后忘记了之前的上下文怎么办？

**A**:
```
1. 重新阅读所有规范
2. 查看项目文档
3. 查看 git 历史（仅查看，不操作）
4. 询问用户补充上下文
5. 不要猜测，确认后再行动
```
