# 验证检查清单

## 1. 修改前检查

### 1.1 文档阅读

```
□ 已阅读相关规范文件
□ 已阅读需求文档
□ 已阅读开发进度计划
□ 理解了任务目标和范围
```

### 1.2 需求确认

```
□ 明确了要实现的功能
□ 明确了验收标准
□ 确认了技术选型（如有疑问已询问）
□ 确认了不涉及范围蔓延
```

### 1.3 数据库变更评估

```
□ 是否需要修改数据库结构？
□ 是否需要创建迁移？
□ 是否会影响现有数据？
□ 是否已评估风险？
```

### 1.4 依赖检查

```
□ 是否需要新增依赖？
□ 新增依赖是否安全？
□ 是否有兼容性问题？
□ 是否已与用户确认？
```

## 2. 修改中检查

### 2.1 开发规范

```
□ 遵循前端/后端规范
□ 遵循命名约定
□ 遵循代码风格
□ 使用类型注解（TypeScript/Python）
```

### 2.2 代码质量

```
□ 代码简洁易懂
□ 有适当的注释（必要时）
□ 避免重复代码
□ 函数/组件职责单一
```

### 2.3 错误处理

```
□ 有适当的错误处理
□ 有适当的输入验证
□ 有边界情况处理
□ 有用户友好的错误提示
```

### 2.4 日志记录

```
□ 关键操作有日志
□ 错误有日志记录
□ 日志级别适当
□ 不记录敏感信息
```

## 3. 修改后检查

### 3.1 代码质量检查

```
□ 运行 GetDiagnostics 无错误
□ 无语法错误
□ 无类型错误
□ 代码符合规范
```

### 3.2 类型检查

**前端**
```
□ npm run type-check 通过
□ 无 TypeScript 错误
□ 类型定义完整
```

**后端**
```
□ python manage.py check 通过
□ 无 Django 检查错误
□ 代码符合 Python 规范
```

### 3.3 测试验证

```
□ 已运行现有测试
□ 测试通过
□ 为新功能添加了测试（如适用）
□ 测试覆盖关键路径
```

### 3.4 构建验证

**前端**
```
□ npm run build 成功
□ 无构建错误
□ 构建产物正常
```

**后端**
```
□ 可以正常启动服务
□ 无启动错误
□ API 端点正常响应
```

## 4. 提交前检查

### 4.1 代码审查

```
□ 调用 TRAE-code-review 检查
□ 审查通过或问题已修复
□ 代码质量符合标准
□ 无明显bug
```

### 4.2 文档更新

```
□ 相关文档已更新
□ API 文档已更新（如适用）
□ README 已更新（如适用）
□ 变更说明清晰
```

### 4.3 提交规范

```
□ 准备好提交信息（按 git.md 规范）
□ 提交信息描述清晰
□ 包含相关 issue 引用（如有）
□ 等待用户手动执行 git 操作
```

## 5. 特殊场景检查

### 5.1 新增数据库表

```
□ Prisma schema 已更新
□ 模型定义完整
□ 索引已添加（如需要）
□ 关系定义正确
□ 迁移文件已创建
□ 已测试迁移
□ 已有数据兼容性已考虑
```

### 5.2 新增 API 端点

```
□ 路由设计符合 RESTful 规范
□ 请求/响应格式定义清晰
□ 认证/授权已实现
□ 输入验证已实现
□ 错误处理已实现
□ API 文档已更新
□ 已测试端点功能
```

### 5.3 新增服务

```
□ 服务职责清晰
□ 接口定义明确
□ 错误处理完善
□ 日志记录充分
□ 可测试性良好
□ 已编写单元测试
```

### 5.4 新增前端组件

```
□ 组件职责单一
□ Props 类型定义完整
□ 有默认值和类型检查
□ 样式使用 CSS Modules
□ 响应式设计已考虑
□ 可访问性已考虑
□ 有组件测试（如适用）
```

### 5.5 修改现有功能

```
□ 理解了原有功能
□ 评估了变更影响
□ 向后兼容性已考虑
□ 相关测试已更新
□ 没有破坏现有功能
□ 用户体验不受负面影响
```

## 6. 发布前检查（可选）

```
□ 所有测试通过
□ 代码审查通过
□ 文档已更新
□ 性能已测试
□ 安全已检查
□ 用户验收测试通过
□ 回滚方案已准备
```

## 7. 单次单功能检查

```
□ 本次只做一件事
□ 功能范围明确
□ 没有混入其他改动
□ 完成后将调用检查
□ 准备好后再继续下一个功能
```

## 8. 快速检查摘要

开发过程中随时可以检查：

```
✅ 修改前：文档 ✓ 需求 ✓ 风险评估 ✓
✅ 修改中：规范 ✓ 质量 ✓ 错误处理 ✓
✅ 修改后：类型 ✓ 测试 ✓ 构建 ✓
✅ 提交前：审查 ✓ 文档 ✓ 准备好 ✓
```
