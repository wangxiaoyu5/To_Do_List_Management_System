# Agent权限规范

## 1. Agent权限边界

### 1.1 明确禁止的操作

> ⚠️ **绝对禁止**：以下操作Agent绝不能执行

```
❌ git操作（所有git命令）
   - git commit
   - git push
   - git pull
   - git checkout
   - git merge
   - git rebase
   - 所有其他git命令
   - 所有git操作必须由用户手动执行

❌ 删除文件（除非用户明确要求）
   - 禁止使用DeleteFile工具，除非用户明确说"删除xxx文件"

❌ 修改用户不允许的文件
   - 禁止修改.gitignore
   - 禁止修改.git目录
   - 禁止修改配置文件（除非用户明确要求）
```

### 1.2 必须明确授权的操作

> ✅ **需要用户明确确认**：以下操作必须得到用户明确授权

```
🔐 创建新文件
   - 必须明确说明要创建什么文件，用户确认后才能创建

🔐 修改项目核心配置
   - package.json
   - settings.py
   - tsconfig.json
   - 等重要配置文件

🔐 运行危险命令
   - 删除命令
   - 数据库修改命令
   - 系统配置命令
```

### 1.3 允许自主执行的操作

> ✅ **可以自主执行**：以下操作无需用户确认即可执行

```
✅ 读取文件
   - Read工具
   - LS工具
   - Glob工具

✅ 检查诊断
   - GetDiagnostics工具

✅ 搜索代码
   - SearchCodebase
   - Grep

✅ 运行安全命令
   - npm run type-check
   - npm run lint
   - python manage.py check
   - 等不修改文件的检查命令
```

## 2. Skill调用权限

### 2.1 Skill调用原则

> 🎯 **Skill调用规则**

```
1. 必须调用与当前任务相关的Skill
2. 按开发流程调用Skill
3. 功能实现后必须调用代码审查Skill
4. 禁止滥用Skill
```

### 2.2 Skill调用权限

```
✅ 可调用的Skill
   - backend-developer
   - backend-development
   - frontend-development
   - frontend-design
   - TRAE-code-review
   - e2e-tester
   - 等开发相关Skill

❌ 禁止自动调用的Skill
   - overnight-dev（除非用户明确要求）
   - 等可能执行大量操作的Skill
```

## 3. 用户交互边界

### 3.1 何时需要询问用户

> ❓ **必须询问用户的场景**

```
1. 技术选型不确定时
   - 选择框架
   - 选择库
   - 选择架构

2. 功能实现方案有多种时
   - 提供2-3个方案
   - 由用户选择

3. 可能影响现有功能时
   - 修改核心逻辑
   - 重构重要模块

4. 需要删除或覆盖文件时
   - 删除文件
   - 覆盖现有文件
```

### 3.2 如何询问用户

> 🗣️ **询问方式**

```
提供明确选项：
- 选项A：描述方案
- 选项B：描述方案
- 选项C：自定义

例如：
> 实现任务筛选功能，有以下方案：
> A. 前端筛选（简单快速）
> B. 后端筛选（性能更好）
> C. 我有其他想法
> 请选择：
```

## 4. 代码修改边界

### 4.1 修改原则

> 📝 **代码修改规则**

```
1. 小范围修改优先
   - 尽量只修改相关文件
   - 避免大范围重构

2. 保持代码风格一致
   - 遵循现有代码规范
   - 使用相同的命名风格

3. 破坏性修改必须确认
   - 修改API接口
   - 修改数据库结构
   - 等可能影响其他模块的修改
```

### 4.2 禁止的代码操作

```
❌ 删除现有功能代码
   - 除非用户明确要求删除

❌ 修改用户明确表示不改动的代码

❌ 引入不必要的依赖
```

## 5. 数据安全边界

### 5.1 敏感数据处理

```
❌ 禁止记录或提交敏感信息
   - 密码
   - API密钥
   - 数据库连接字符串
   - .env文件内容

⚠️ 环境变量处理
   - .env文件只作为读取参考
   - 绝不提交到代码库
   - 提供.env.example模板
```

## 6. 错误处理边界

### 6.1 出错时的行为

```
1. 出错后立即停止
   - 不继续执行可能错误的操作
   - 向用户报告错误

2. 提供有用的错误信息
   - 错误原因
   - 可能的解决方案

3. 不隐藏错误
   - 不假装成功
   - 如实报告问题
```

## 7. 任务范围边界

### 7.1 任务范围判断

```
✅ 接受的任务
   - 用户明确描述的功能
   - 修复明确报告的bug
   - 按计划进行的开发

❌ 超出范围的任务
   - 用户没要求的额外功能
   - 过度优化
   - 范围蔓延
```

## 8. 总结：权限边界检查清单

```
执行任何操作前，请检查：
□ 这个操作是用户明确要求的吗？
□ 这个操作在我的权限范围内吗？
□ 这个操作安全吗？
□ 需要询问用户吗？
□ 会产生什么后果？
□ 用户会同意吗？
```
