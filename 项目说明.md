## 项目概述

### 🎯 项目介绍
这是一个功能完善的Web端待办事项管理系统，采用前后端分离架构，支持多用户登录认证，帮助用户高效管理日常任务。

### ✨ 核心功能
- 用户注册/登录认证
- 任务的增删改查
- 任务分类管理
- 任务标签管理
- 任务完成状态跟踪
- 任务搜索与筛选
- 数据导出导入

## 技术架构

### 🛠️ 技术栈

#### 前端技术
- **框架**: React 18 + TypeScript
- **UI库**: Ant Design 5.x
- **状态管理**: Redux Toolkit
- **路由**: React Router v6
- **构建工具**: Vite
- **HTTP客户端**: Axios
- **测试**: Vitest

#### 后端技术
- **框架**: Python Django 4.x
- **API**: Django REST Framework
- **ORM**: Prisma
- **数据库**: SQLite (开发环境)
- **认证**: JWT (djangorestframework-simplejwt)
- **CORS**: django-cors-headers

### 📊 数据库设计

#### Prisma Schema (`backend/prisma/schema.prisma`)
```prisma
model User {
  id        Int      @id @default(autoincrement())
  username  String   @unique
  email     String   @unique
  password  String
  tasks     Task[]
  categories Category[]
  tags      Tag[]
}

model Task {
  id          Int      @id @default(autoincrement())
  title       String
  description String?
  priority    String   @default("medium")
  completed   Boolean  @default(false)
  dueDate     DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  user        User     @relation(fields: [userId], references: [id])
  userId      Int
  category    Category? @relation(fields: [categoryId], references: [id])
  categoryId  Int?
  tags        Tag[]
}

model Category {
  id        Int      @id @default(autoincrement())
  name      String
  color     String
  user      User     @relation(fields: [userId], references: [id])
  userId    Int
  tasks     Task[]
}

model Tag {
  id        Int      @id @default(autoincrement())
  name      String
  color     String
  user      User     @relation(fields: [userId], references: [id])
  userId    Int
  tasks     Task[]
}
```

## 后端开发流程

### 1️⃣ 项目初始化
```bash
# 创建Django项目
django-admin startproject todo_project
cd todo_project

# 创建API应用
python manage.py startapp api
```

### 2️⃣ 数据库模型设计
1. 在 `prisma/schema.prisma` 中定义数据模型
2. 生成数据库迁移: `npx prisma migrate dev`
3. 生成Prisma客户端: `npx prisma generate`

### 3️⃣ Django模型创建
在 `backend/api/models.py` 中创建Django模型，与Prisma保持一致。

### 4️⃣ 序列化器开发
在 `backend/api/serializers.py` 中创建DRF序列化器，用于：
- 请求数据验证
- 响应数据格式化
- 模型数据和JSON的转换

### 5️⃣ API视图开发
在 `backend/api/views.py` 中使用ViewSet创建RESTful API。

### 6️⃣ URL路由配置
在 `backend/api/urls.py` 中注册API路由。

### 7️⃣ 认证配置
使用 `djangorestframework-simplejwt` 配置JWT认证。

### 8️⃣ 后端启动
```bash
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

## 前端开发流程

### 1️⃣ 项目初始化
```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
```

### 2️⃣ 安装依赖
```bash
npm install antd @ant-design/icons react-router-dom
npm install @reduxjs/toolkit react-redux axios
npm install -D vitest @testing-library/react @testing-library/jest-dom
```

### 3️⃣ 组件开发
1. **页面组件**: 认证页面、主页面
2. **业务组件**: 任务列表、任务表单、分类管理、标签管理
3. **共享组件**: 可复用的基础组件

### 4️⃣ 状态管理
使用Redux Toolkit管理：
- 认证状态（用户信息、Token）
- 数据状态（任务列表、分类、标签）

### 5️⃣ API集成
在 `frontend/src/services/api.ts` 中封装Axios，统一处理：
- 请求拦截器（添加Token）
- 响应拦截器（错误处理）
- 所有API调用方法

### 6️⃣ 路由配置
使用React Router配置页面路由：
- `/` - 登录/注册页面
- `/app` - 主应用页面

### 7️⃣ 前端启动
```bash
cd frontend
npm install
npm run dev
```

## 功能模块详解

### 🔐 用户认证模块

#### 注册流程
1. 用户填写用户名、邮箱、密码
2. 前端验证表单数据
3. 发送 `POST /api/auth/register` 请求
4. 后端创建用户，密码bcrypt加密
5. 返回JWT Token
6. 前端存储Token，跳转到主页面

#### 登录流程
1. 用户输入用户名/邮箱 + 密码
2. 发送 `POST /api/auth/login` 请求
3. 后端验证凭证
4. 返回JWT Token (access + refresh)
5. 前端存储Token
6. 后续请求在Header中携带: `Authorization: Bearer ${token}`

#### 登出流程
1. 清除本地Token存储
2. 清除Redux状态
3. 跳转到登录页面

### 📋 任务管理模块

#### 任务列表展示
- 从API获取用户所有任务
- 根据状态筛选（全部/已完成/未完成）
- 卡片式展示，包含标题、描述、优先级、分类、标签、截止日期
- 支持搜索和多条件筛选

#### 创建任务
1. 点击"添加任务"按钮
2. 打开任务表单弹窗
3. 填写任务信息
4. 发送 `POST /api/tasks` 请求
5. 刷新任务列表

#### 编辑任务
1. 点击任务卡片的编辑按钮
2. 预填充表单数据
3. 修改后保存
4. 发送 `PUT /api/tasks/:id` 请求

#### 删除任务
1. 点击删除按钮
2. 二次确认弹窗
3. 发送 `DELETE /api/tasks/:id` 请求

#### 标记完成
1. 点击复选框
2. 发送 `PATCH /api/tasks/:id/toggle` 请求
3. 视觉上标记为完成状态

### 🏷️ 分类与标签模块

#### 分类管理
- 树形结构展示分类
- 创建、编辑、删除分类
- 每个分类可以设置颜色
- 任务关联到分类

#### 标签管理
- 标签列表展示
- 创建、编辑、删除标签
- 每个标签可以设置颜色
- 任务可以添加多个标签

### 🔍 搜索与筛选

#### 搜索
- 实时搜索任务标题和描述
- 输入时即时过滤

#### 筛选
- 按分类筛选
- 按标签筛选
- 按优先级筛选
- 按完成状态筛选
- 多条件组合
