# 待办事项管理系统 - 需求文档

## 1. 项目概述

### 1.1 项目目标
开发一个功能完善、界面美观的Web端待办事项管理系统，支持多用户登录认证，帮助用户高效管理日常任务。

### 1.2 目标用户
- 需要管理工作任务的职场人士
- 需要规划学习计划的学生
- 需要安排日常事务的普通用户

---

## 2. 功能需求

### 2.1 用户认证模块

#### 2.1.1 用户注册
- 用户可以注册新账号
- 包含用户名、邮箱、密码
- 密码加密存储
- 邮箱验证（可选）

#### 2.1.2 用户登录
- 支持用户名/邮箱 + 密码登录
- JWT Token认证
- 登录状态保持
- 退出登录功能

#### 2.1.3 用户信息管理
- 查看和编辑个人资料
- 修改密码
- 用户数据隔离（每个用户只能看到自己的任务）

### 2.2 任务管理模块

#### 2.2.1 创建任务
- 用户可以创建新的待办事项
- 任务包含以下字段：
  - 标题（必填）
  - 描述（可选）
  - 分类（可选）
  - 标签（可选，多个）
  - 优先级（必填，默认中等）
  - 截止日期（可选）
  - 创建时间（自动生成）
  - 创建者（自动关联当前用户）

#### 2.2.2 查看任务
- 以列表形式展示当前用户的所有任务
- 支持卡片视图和列表视图切换
- 显示任务的基本信息和状态

#### 2.2.3 编辑任务
- 用户可以修改已有任务的所有字段
- 保留修改历史记录

#### 2.2.4 删除任务
- 用户可以删除不再需要的任务
- 删除前提示确认

#### 2.2.5 任务完成状态跟踪
- 支持标记任务为"已完成"或"未完成"
- 已完成任务显示视觉区分（灰色、划线）
- 显示完成时间
- 统计完成率

### 2.3 分类与标签模块

#### 2.3.1 分类管理
- 用户可以创建、编辑、删除自定义分类
- 预设分类：工作、个人、学习、其他
- 每个任务只能属于一个分类
- 分类属于用户私有

#### 2.3.2 标签管理
- 用户可以创建、编辑、删除自定义标签
- 一个任务可以添加多个标签
- 支持颜色自定义
- 标签属于用户私有

### 2.4 优先级模块

#### 2.4.1 优先级级别
- 高优先级（红色）
- 中优先级（黄色）
- 低优先级（绿色）

#### 2.4.2 优先级展示
- 在任务卡片上显示优先级颜色标识
- 支持按优先级排序

### 2.5 截止日期与提醒模块

#### 2.5.1 截止日期
- 支持设置任务截止日期和时间
- 即将过期的任务高亮显示
- 已过期的任务红色警告

#### 2.5.2 提醒功能
- 截止日期前的任务在界面上突出显示

### 2.6 搜索与筛选模块

#### 2.6.1 搜索功能
- 支持按关键词搜索任务标题和描述
- 实时搜索结果

#### 2.6.2 筛选功能
- 按分类筛选
- 按标签筛选
- 按优先级筛选
- 按完成状态筛选
- 多条件组合筛选

#### 2.6.3 排序功能
- 按截止日期排序（升序/降序）
- 按创建日期排序（升序/降序）
- 按优先级排序

### 2.7 数据导出/导入模块

#### 2.7.1 数据导出
- 支持导出任务数据为JSON格式
- 支持导出任务数据为CSV格式
- 导出文件包含用户所有任务、分类、标签

#### 2.7.2 数据导入
- 支持从JSON文件导入任务数据
- 支持从CSV文件导入任务数据
- 导入前数据验证
- 导入冲突处理

## 3. 非功能需求

### 3.1 界面设计
- 简洁现代的UI风格
- 响应式设计，适配桌面和移动设备
- 清晰的视觉层次和交互反馈

### 3.2 性能要求
- 页面加载时间 < 2秒
- 任务列表流畅滚动
- 操作响应及时
- 支持1000+任务数据量

### 3.3 数据安全
- 用户密码加密存储（bcrypt）
- JWT Token认证
- 用户数据隔离
- HTTPS传输（生产环境）

### 3.4 兼容性
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端浏览器适配

## 4. 技术架构

### 4.1 技术栈

#### 前端
- **框架**：React 18+
- **UI组件库**：Ant Design 5.x
- **状态管理**：Redux Toolkit + RTK Query
- **构建工具**：Vite
- **样式**：CSS Modules / Tailwind CSS
- **路由**：React Router v6
- **HTTP客户端**：Axios

#### 后端
- **框架**：Python Django 4.x + Django REST Framework
- **数据库**：PostgreSQL
- **ORM**：Prisma
- **认证**：JWT (djangorestframework-simplejwt)
- **CORS**：django-cors-headers

### 4.3 API接口设计

#### 认证接口
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户退出
- `GET /api/auth/user` - 获取当前用户信息

#### 任务接口
- `GET /api/tasks` - 获取任务列表
- `POST /api/tasks` - 创建任务
- `GET /api/tasks/:id` - 获取任务详情
- `PUT /api/tasks/:id` - 更新任务
- `DELETE /api/tasks/:id` - 删除任务
- `PATCH /api/tasks/:id/complete` - 标记任务完成/未完成

#### 分类接口
- `GET /api/categories` - 获取分类列表
- `POST /api/categories` - 创建分类
- `PUT /api/categories/:id` - 更新分类
- `DELETE /api/categories/:id` - 删除分类

#### 标签接口
- `GET /api/tags` - 获取标签列表
- `POST /api/tags` - 创建标签
- `PUT /api/tags/:id` - 更新标签
- `DELETE /api/tags/:id` - 删除标签

#### 数据导入导出接口
- `GET /api/export/json` - 导出JSON
- `GET /api/export/csv` - 导出CSV
- `POST /api/import/json` - 导入JSON
- `POST /api/import/csv` - 导入CSV
