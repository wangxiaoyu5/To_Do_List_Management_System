#### **第1-11行：TimeStampedModel 基类**

```python
from django.db import models
from django.conf import settings
from django.utils import timezone

class TimeStampedModel(models.Model):
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        abstract = True
```

**📖 逐行解析：**

| 行号 | 代码 | 作用说明 | 为什么这么写？ |
|------|------|----------|----------------|
| 1 | `from django.db import models` | 导入Django的ORM模型模块 | Django ORM是核心，所有数据库操作都通过它 |
| 3 | `from django.utils import timezone` | 导入时区处理工具 | Django推荐使用`timezone`而不是`datetime`，处理时区更安全 |
| 6 | `class TimeStampedModel(models.Model):` | 定义一个抽象基类 | **抽象基类模式**：让其他模型继承它，避免重复写`created_at`和`updated_at`字段 |
| 7 | `created_at = models.DateTimeField(auto_now_add=True)` | 创建时间字段 | `auto_now_add=True`：创建时自动设置为当前时间，之后不改变 |
| 8 | `updated_at = models.DateTimeField(auto_now=True)` | 更新时间字段 | `auto_now=True`：每次保存时自动更新为当前时间 |
| 10-11 | `class Meta: abstract = True` | 声明为抽象模型 | 抽象模型不会在数据库中创建表，只用于继承 |

**✅ 如何修改：**
- 如果想添加其他通用字段（如`created_by`），可以在这个基类中添加

#### **第14-28行：Category（分类）模型**

```python
class Category(TimeStampedModel):
    name = models.CharField(max_length=100)
    color = models.CharField(max_length=20, default='#1890ff')
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name='categories'
    )

    class Meta:
        ordering = ['-created_at']
        unique_together = ['user', 'name']

    def __str__(self):
        return self.name
```

**📖 逐行解析：**

| 行号 | 代码 | 作用说明 | 为什么这么写？ |
|------|------|----------|----------------|
| 14 | `class Category(TimeStampedModel):` | 继承TimeStampedModel | 自动获得`created_at`和`updated_at`字段 |
| 15 | `name = models.CharField(max_length=100)` | 分类名称字段 | `CharField`：字符串类型，必须指定`max_length` |
| 16 | `color = models.CharField(max_length=20, default='#1890ff')` | 分类颜色字段 | `default`：提供默认值，蓝色`#1890ff`是Ant Design主题色 |
| 17-21 | `user = models.ForeignKey(...)` | 关联用户 | **外键关系**：每个分类属于一个用户 |
| 18 | `settings.AUTH_USER_MODEL` | 使用Django用户模型 | 引用`settings`中的配置，而不是硬编码 |
| 19 | `on_delete=models.CASCADE` | 级联删除 | 用户删除时，他的所有分类也自动删除，避免孤儿数据 |
| 20 | `related_name='categories'` | 反向关系名称 | 通过`user.categories`可以获取该用户的所有分类 |
| 24 | `ordering = ['-created_at']` | 默认排序 | `-`表示降序，新创建的分类排在前面 |
| 25 | `unique_together = ['user', 'name']` | 联合唯一约束 | 同一用户下不能有两个同名分类，数据一致性 |
| 27-28 | `def __str__(self): return self.name` | 字符串表示 | 在Admin后台和调试时显示分类名称而不是对象ID |

**✅ 如何修改：**
- 想让分类可以嵌套？添加`parent = models.ForeignKey('self', null=True, blank=True, ...)`
- 想添加描述字段？添加`description = models.TextField(blank=True)`

#### **第48-90行：Task（任务）模型**

```python
class Task(TimeStampedModel):
    PRIORITY_CHOICES = [
        ('HIGH', 'High'),
        ('MEDIUM', 'Medium'),
        ('LOW', 'Low'),
    ]

    title = models.CharField(max_length=200)
    description = models.TextField(blank=True, null=True)
    priority = models.CharField(
        max_length=10,
        choices=PRIORITY_CHOICES,
        default='MEDIUM',
        db_index=True
    )
    due_date = models.DateTimeField(null=True, blank=True, db_index=True)
    completed = models.BooleanField(default=False, db_index=True)
    completed_at = models.DateTimeField(null=True, blank=True)
    
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name='tasks'
    )
    category = models.ForeignKey(
        Category,
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name='tasks'
    )
    tags = models.ManyToManyField(Tag, through='TaskTag', related_name='tasks')

    class Meta:
        ordering = ['-created_at']

    def __str__(self):
        return self.title

    def mark_completed(self):
        self.completed = True
        self.completed_at = timezone.now()
        self.save()
```

**📖 逐行解析：**

| 行号 | 代码 | 作用说明 | 为什么这么写？ |
|------|------|----------|----------------|
| 49-53 | `PRIORITY_CHOICES = [...]` | 优先级选项列表 | **CHOICES模式**：第一个值存数据库，第二个值显示给用户 |
| 57-62 | `priority = models.CharField(...)` | 优先级字段 | `choices`：限制只能选指定值，防止非法输入 |
| 61 | `db_index=True` | 数据库索引 | 按优先级查询很常见，加索引提升查询速度 |
| 63 | `due_date = models.DateTimeField(null=True, blank=True, ...)` | 截止日期 | `null=True`：数据库中可以为空；`blank=True`：表单中可以不填 |
| 72-78 | `category = models.ForeignKey(...)` | 关联分类 | `on_delete=models.SET_NULL`：删除分类时，任务的分类设为空，而不是删除任务 |
| 79 | `tags = models.ManyToManyField(Tag, through='TaskTag', ...)` | 多对多标签 | `through='TaskTag'`：使用中间表，保存额外信息（如创建时间） |
| 87-90 | `def mark_completed(self): ...` | 自定义方法 | **胖模型模式**：把业务逻辑放在模型里，代码更清晰、可复用 |

**✅ 如何修改：**
- 想添加"提醒时间"字段？添加`reminder_at = models.DateTimeField(null=True, blank=True)`
- 想添加优先级图标？在`PRIORITY_CHOICES`中可以添加图标信息，或在前端处理
